# Jobs to Be Done Analysis: How Many Stars

## The Three Questions

### 1. Who pays to solve this today?

Nobody. GitHub exposes star counts in its own interface for free. Third-party GitHub analytics tools (Star History, ghstats.net, repobeats.axiom.co) are free. The gap this tool closes — "one page that totals my stars across repos, sorted by count" — is a minor convenience, not a pain point anyone has demonstrated willingness to pay to resolve. The vision document already names this: "This isn't a crisis. It's a quality-of-life gap that was worth a few hours to close."

**Score: 0/1**

### 2. What do they pay?

Nothing. GitHub analytics is a free-tier category without exception. The closest paid adjacent products (GitHub Advanced Security, Paddle analytics dashboards) serve different use cases entirely. No developer is paying a subscription to see their aggregate star count.

**Score: 0/1**

### 3. Why would they switch to this?

No paid incumbent to switch from. The vision document identifies a potential productization path — parameterize the username, add caching, deploy to a persistent URL — but even that path lands in a free-tool category competing with Star History and similar tools, all of which are also free. UX advantage on a free personal utility does not create a switching argument in a revenue context.

**Score: 0/1**
